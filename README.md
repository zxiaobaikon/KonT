## 📖 简介

本项目是一个**无需任何用户认证**的 Web 工具，提供以下核心能力：

- 📂 **文件管理**：浏览、上传、下载、删除、重命名、复制/移动、新建文件/文件夹
- ✏️ **在线编辑器**：支持 PHP、HTML、CSS、JS 等多种语法高亮，带搜索/替换/跳转
- 📊 **系统监控**：实时显示系统负载、CPU 使用率、内存和磁盘占用（基于 Linux 命令）
- 🔍 **全局搜索**：按文件名快速定位文件（基于 `find` 命令）
- 🖱️ **右键菜单**：所有操作均可通过右键快速完成
- 📱 **响应式设计**：完美适配桌面端与移动端

> ⚠️ **安全提示**：本工具默认无任何身份验证，**请勿直接部署在公网环境**。建议在本地网络、开发环境或通过 HTTP 基础认证使用。

## 📦 安装与配置

### 1. 环境要求
- PHP 7.4 或更高版本（推荐 8.0+）
- Linux / Unix 操作系统（监控命令依赖 `top`、`free`、`df`）
- Web 服务器（Apache / Nginx）

### 2. 安装步骤
1. 将 `webt.php` 文件下载到您的网站目录（例如 `/var/www/html/`）。
2. 使用文本编辑器打开文件，找到配置区（约第 7-9 行）：
   ```php
   // ---------- 配置区 ----------
   // 设置你的网站根目录（所有文件操作将限制在此目录内）
   define('ROOT_PATH', '/var/www/html');  // 请修改为你的实际路径
   // -----------------------------
   将 /var/www/html 替换为您希望管理的目录的绝对路径。

确保 Web 服务器对 ROOT_PATH 及其子目录具有读写权限（至少需要读权限）。

通过浏览器访问 http://您的域名/webt.php 即可使用。

🔒 安全建议
由于本工具无任何用户认证，请务必注意：

✅ 仅在本地网络或开发环境使用

✅ 通过 Web 服务器（如 Nginx）添加HTTP 基础认证

✅ 或在代码顶部添加 IP 白名单验证

❌ 不要将 webt.php 暴露在公网，除非您已自行实现认证逻辑

🐛 常见问题
1. 监控数据不更新 / 显示为 0
检查服务器是否禁用了 shell_exec 函数。

确认 top、free、df 命令是否可用。

2. 上传文件失败
检查目标目录是否有写权限。

查看 PHP 配置中的 upload_max_filesize 和 post_max_size。

3. 编辑器无法保存
检查文件权限，确保 Web 用户有写入权限。

查看 PHP 错误日志获取详细原因。

4. 搜索功能无结果
确认 find 命令可用，且 ROOT_PATH 下存在文件。
